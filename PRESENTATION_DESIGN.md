# CostIQ Presentation Design

Дата: 2026-05-22
Статус: обязательный дизайн-контекст для презентаций CostIQ / Agent Factory

## Назначение

Этот файл фиксирует стиль презентаций CostIQ, чтобы AI-агенты и разработчики могли повторять утвержденный визуальный подход без ручного пересоздания правил.

Базовый эталон: слайд по супер-навыку bot-manager от 2026-05-21:

- PPTX: /home/ClawdCostIQ/workspace/exports/bot_manager_super_skill_slide_2026-05-21.pptx
- PNG: /home/ClawdCostIQ/workspace/exports/bot_manager_super_skill_slide_2026-05-21.png

## Общий Стиль

Название стиля: FSK Dark Copper.

Формат:

- 16:9;
- темный холст;
- медный акцент;
- плотная управленческая структура;
- один главный вывод на слайд;
- минимум декоративности;
- максимум смысловой читаемости.

## Палитра

Базовые цвета:

- фон: #141414;
- карточки и рабочие блоки: #1F1F1F;
- вторичные поверхности: #242424;
- основной текст: #F5F5F5;
- вторичный текст: #B8B8B8;
- приглушенный текст: #8A8A8A;
- медный акцент: #E84E0E;
- светлый медный акцент: #FF8A1E;
- линии: #343434;
- риск / ошибка: #FF6B5F;
- успех / готово: #4FC37A.

Градиент допустим только как медный акцент в небольших элементах. Не делать слайд градиентным фоном.

## Типографика

- Шрифт: Arial.
- Заголовок: крупный, но не декоративный.
- Подзаголовок: короткий управленческий смысл.
- Текстовые блоки: компактные, без длинных абзацев.
- Letter spacing: 0.
- Не использовать слишком мелкий текст для ключевых выводов.

## Композиция

Рекомендуемые схемы:

- главный вывод сверху + 3 карточки;
- процессная цепочка;
- 2 колонки "было / стало";
- матрица 2x2 или 3x3;
- dashboard со счетчиками;
- карточка агента + схема связей;
- дорожная карта этапов.

Запрещено:

- маркетинговая hero-страница;
- перегруженный текстовый документ на слайде;
- случайные иконки без функции;
- декоративные картинки вместо смысла;
- разнородные стили в одной презентации.

## Карточки

Карточка должна содержать:

- короткий заголовок;
- 2-4 факта;
- одну метрику или статус, если она нужна;
- медный или серый акцент.

Радиус карточек: до 8 px.

Не вкладывать карточки в карточки.

## Иконки И Схемы

Иконки использовать только для ориентации:

- агент;
- навык;
- заявка;
- файл;
- проверка;
- результат;
- acceptance;
- риск.

Схемы должны объяснять процесс, а не украшать слайд.

## Содержание Слайда

Каждый слайд должен отвечать на один вопрос:

- что сделали;
- зачем это нужно;
- какой эффект;
- что входит в контур;
- что дальше.

Для Agent Factory и bot-manager использовать цепочку:

заявка -> паспорт -> Telegram modes -> навыки -> role-based меню -> Mini App cards -> allowlist -> Bot-to-Bot -> антициклы -> acceptance -> эксплуатация.

## Для Слайдов По Панели

Показывать панель как продуктовый контур:

- заявка;
- файл;
- обработка навыком;
- результат;
- вопрос / доработка;
- новая версия;
- принятие;
- история;
- статистика;
- инструменты и калькуляторы.

Ключевой тезис: панель уже не MVP, а рабочий контур обработки заявок и инструментов САУ.

## Для Слайдов По ИИ-Рабочему Столу

Рекомендуемая структура:

- слева: контекст объекта / файла / подрядчика;
- центр: ИИ-анализ или диалог;
- справа: быстрые калькуляторы и действия;
- снизу: создать заявку, сделать документ, сохранить в объект.

Не показывать ИИ-рабочий стол как обычный чат. Это рабочее место САУ.

## Для Слайдов По Калькуляторам

Показывать не "онлайн-калькулятор", а быстрый инструмент САУ/ПТО:

- НДС;
- проценты;
- отклонения КП/ПС/ГТ/медианы;
- цена за единицу;
- обратный расчет объема;
- ПТО-расчеты;
- копирование результата;
- сохранение в объект или заявку.

## Acceptance Для Презентации

Перед выдачей PPTX/PNG проверить:

- формат 16:9;
- темный холст и медный акцент соответствуют FSK Dark Copper;
- один главный вывод читается за 5 секунд;
- нет мелкого текста в ключевых местах;
- карточки не перегружены;
- слайд не выглядит как лендинг;
- нет случайных градиентных украшений;
- все цифры и факты имеют источник или понятное происхождение;
- если есть файл на выдачу, он сохранен в /home/ClawdCostIQ/workspace/exports.
